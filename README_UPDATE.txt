
============================================================
一旦テスト送信する場合（推奨）
============================================================

1. Apps Scriptへ apps_script_backend.gs を貼り付けて保存します。
2. 関数選択欄から authorizeWeeklyScheduleEmail を1回実行し、権限を許可します。
3. 関数選択欄から sendNextWeekScheduleTestEmail を実行します。
4. 次の1アドレスだけにテストメールが送信されます。

   ito.shin.z0@s.mail.nagoya-u.ac.jp

件名の先頭には「【テスト】」が付きます。
このテスト関数を実行しても、users_movpe7@googlegroups.com には送信されず、
毎週金曜日の自動送信済み判定にも影響しません。

注意：sendNextWeekScheduleEmail は本番メーリングリストへ送る関数です。
テスト時には選択しないでください。

MOroom-reservation 毎週金曜日・次週予定メール 自動送信版

ファイル名は変更していません。ZIP内は従来と同じ6ファイルです。
今回の機能追加は apps_script_backend.gs に統合しています。

==================================================
追加した自動メールの仕様
==================================================
- 毎週金曜日の朝8:00前後（日本時間）に自動実行
- 翌週の月曜日～日曜日の予約を取得
- 全装置の予定を週間表としてメール本文に表示
- 付帯設備メンテ・ガス交換予定も同じメールに表示
- users_movpe7@googlegroups.com へ送信
- 件名：
  【MO連絡網】次週のMOVPE使用予定（7/27～8/2）
- 同じ内容をPDFで添付
- 予約がない場合も「次週の予約はありません」と送信
- 同じ対象週への自動メールの二重送信を防止
- 一時的に作成するGoogleドキュメントはPDF変換後に自動でゴミ箱へ移動

==================================================
1. Google Apps Scriptのコードを更新
==================================================
1) 予約用スプレッドシートを開く
2) 拡張機能 → Apps Script
3) 既存コードを apps_script_backend.gs の内容で全置換
4) 自動保存を確認
5) setup を1回実行
   ※既存の予約データは削除されません

==================================================
2. 自動メール用の権限を許可
==================================================
1) Apps Script上部の関数選択欄から
   authorizeWeeklyScheduleEmail
   を選択
2) 「実行」を押す
3) 権限確認画面で以下を許可
   - Googleスプレッドシートの参照
   - メール送信
   - Googleドキュメントの作成
   - Googleドライブ上の一時ファイル操作

この関数ではメールを送信しません。
確認用PDFを一時生成し、直後にゴミ箱へ移動します。

==================================================
3. 毎週金曜日のトリガーを設定
==================================================
1) 関数選択欄から
   installWeeklyScheduleEmailTrigger
   を選択
2) 「実行」を押す
3) 実行ログに
   「毎週金曜日8時前後の自動送信トリガーを設定しました。」
   と表示されれば完了
4) Apps Script左側の時計アイコン「トリガー」を開き、以下を確認
   - 関数：sendNextWeekScheduleEmailScheduled
   - イベントのソース：時間主導型
   - 曜日：金曜日
   - 時刻：8時台

installWeeklyScheduleEmailTrigger は、同じ自動送信トリガーを先に削除してから
1件だけ作り直すため、複数回実行しても重複トリガーになりません。

==================================================
4. テスト送信（任意）
==================================================
方法A：メーリングリストへ実際に送信
- sendNextWeekScheduleEmail を実行

方法B：自分のメールへテスト送信
1) apps_script_backend.gs 上部付近の次の設定に自分のメールアドレスを入力

   const WEEKLY_EMAIL_TEST_RECIPIENT = '自分のメールアドレス';

2) sendNextWeekScheduleTestEmail を実行

テスト送信は、金曜日の自動送信済み判定には影響しません。

==================================================
5. Webアプリを新バージョンで再デプロイ
==================================================
Apps Scriptのコードを全置換した後は、予約サイト側も同じ最新版を使うために、
次の手順で既存Webアプリを更新してください。

1) デプロイ → デプロイを管理
2) 対象デプロイの鉛筆マーク
3) バージョン → 新バージョン
4) デプロイ

既存デプロイを更新すれば、通常は /exec のURLは変わりません。

==================================================
6. GitHub側について
==================================================
今回の自動メール機能だけであれば、GitHub側の画面ファイルの変更は不要です。
ZIPには現在の同名ファイル一式を入れてありますが、必須更新は次のファイルです。

- apps_script_backend.gs
- README_UPDATE.txt（手順確認用）

==================================================
管理用関数
==================================================
authorizeWeeklyScheduleEmail
  自動メールに必要な権限を確認します。メールは送りません。

installWeeklyScheduleEmailTrigger
  毎週金曜8時前後のトリガーを1件作成します。

sendNextWeekScheduleEmail
  メーリングリストへ手動でテスト送信します。

sendNextWeekScheduleTestEmail
  WEEKLY_EMAIL_TEST_RECIPIENT に設定したアドレスへテスト送信します。

removeWeeklyScheduleEmailTrigger
  金曜日の自動送信トリガーを削除します。

==================================================
注意事項
==================================================
- トリガーの実行時刻は厳密な8:00ではなく、通常は8:00前後になります。
- WebアプリをデプロイしたGoogleアカウントが、
  users_movpe7@googlegroups.com へ投稿できる必要があります。
- Googleグループ側で外部投稿やメンバー投稿が制限されている場合は、
  デプロイしたアカウントを投稿可能なメンバーにしてください。
- 送信元は、トリガーを作成したGoogleアカウントです。
- PDF生成のため、一時Googleドキュメントが作られますが、自動でゴミ箱へ移動します。
- メール本文はHTML週間表、添付はA4横向きPDFです。
