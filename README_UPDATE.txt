MOroom予約サイト 送信先変更版（v20）

============================================================
今回追加した機能
============================================================
サイト上で次の情報が登録・変更・削除された直後に、
mousers@googlegroups.com へ自動メールを送信します。

1. MOVPE予約フォームのメンテ
   - 原料交換
   - 重故障
   - 除害停止
   - 定常メンテ

2. 付帯設備メンテ・ガス交換予定
   - 付帯設備メンテ
   - ガス交換

3. 管理室からの連絡
   - 通常
   - 重要
   - 緊急

通常の「エピ」予約だけではメールを送りません。

変更時のメールには「変更前」と「変更後」を掲載します。
削除時のメールには削除した内容を掲載します。
件名には必ず「MO連絡網」が入るため、サイトのメール内容欄にも表示されます。

例：
【MO連絡網】【MOVPEメンテ・登録】MOVPE #7
【MO連絡網】【ガス交換・変更】NH3ボンベ交換
【MO連絡網】【管理室連絡（緊急）・登録】除害装置停止

============================================================
安全仕様
============================================================
メール送信だけに失敗しても、登録・変更・削除の処理は取り消しません。
画面には次のような警告を返します。

「通知メールの送信に失敗しました。登録内容は保存されています。」

これにより、メールエラー後の再登録による予約重複を防ぎます。

============================================================
反映手順
============================================================
今回、GitHub側のファイル更新は不要です。
更新するのは Google Apps Script のみです。

1. 予約用スプレッドシートを開く
2. 拡張機能 → Apps Script
3. 既存コードを apps_script_backend.gs の内容で全置換
4. 自動保存を確認
5. 関数選択欄から authorizeInstantNotificationEmail を1回実行
6. 権限確認が出た場合はメール送信権限を許可
7. デプロイ → デプロイを管理
8. 対象デプロイの鉛筆マーク
9. バージョンを「新バージョン」にしてデプロイ

すでに週間予定メールのテスト送信が成功している場合、
メール送信権限は許可済みです。それでも
authorizeInstantNotificationEmail を実行して確認して構いません。

setup は今回の機能追加には必須ではありません。
実行しても既存予約は削除されません。

============================================================
個人アドレスへの安全なテスト
============================================================
関数選択欄から次を実行してください。

sendInstantNotificationTestEmail

次のアドレスだけへ、サンプルのメンテ登録メールを送信します。

ito.shin.z0@s.mail.nagoya-u.ac.jp

このテスト関数では、予約・管理室連絡・付帯設備予定を登録しません。
mousers@googlegroups.com にも送信しません。

テスト成功後、サイトから実際にメンテ等を登録すると、
mousers@googlegroups.com へ自動通知されます。

============================================================
既存の週間予定メール
============================================================
毎週金曜日8時前後の次週予定メール機能はそのまま残しています。
既存のトリガーを作り直す必要はありません。

以下の関数も従来どおり使用できます。
- authorizeWeeklyScheduleEmail
- installWeeklyScheduleEmailTrigger
- sendNextWeekScheduleTestEmail
- sendNextWeekScheduleEmailScheduled

============================================================
注意
============================================================
- Webアプリを「新バージョン」で再デプロイしないと、サイトからの登録時通知は動きません。
- 自動保存だけでは公開Webアプリへ反映されません。
- 送信元はWebアプリをデプロイしたGoogleアカウントです。
- そのアカウントが mousers@googlegroups.com へ投稿できる必要があります。
- Google Groups側の投稿権限で拒否された場合も、登録内容自体は保存されます。
